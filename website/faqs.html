Frequently Asked Questions About emDEX
Q. Why don’t all of my markers appear in the emDEX window?
A. If it’s a book you are trying to view, you must start emDEX with the focus on the book file. If you are running the demo version, you will only see the first 100 markers.
Q. When I tried to start emDEX, why did I receive a warning message that the emDEX server is not active?
A. When FrameMaker® is started, the emDEX server icon should appear in the tooltray. If it does not appear, either restart FrameMaker®, or start the emDEX server from Start>Programs>emDEX server.
Q. Why is all the imported text highlighted when I click on an index entry that is contained in imported text?
A. FrameMaker identifies index entries in text insets the same way.
Q. Why when I deleted one entry in emDEX other entries were deleted as well?
A. If within one marker you have several entries, deleting one entry will delete all other entries within that marker.
Q. Why is the page number displayed in emDEX different from where the marker is?
A. The marker page is taken from the beginning of the enclosing paragraph.
Q. Why did all the emDEX entries disappear when I closed a file?
A. emDEX will only display markers for open documents. Automatically, emDEX will open all files in the book. Closing a file will remove all the entries in emDEX for that particular file.
Q. I have Real-time updating turned off. When I click on an existing marker why can’t emDEX find it?
A. You have to refresh the emDEX window before searching for an entry, if changes have been made to the document.
Q. Why, when I try to open the emDEX window, does nothing appear?
A. Increase the amount of virtual memory available. This can be accessed through the System option in the Control Panel. Some versions of Windows also allow you to let the Operating System manage the Virtual Memory settings, which is recommended.
Q. What versions of FrameMaker® will emDEX work with?
A. Versions 5.5, 5.5.6, 6.0, 7.0, 7.1, 7.2, and 8.0
